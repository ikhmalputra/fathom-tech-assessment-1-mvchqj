Technical Assessment - Front-End Developer

This is a simple technical assessment designed to gauge your technical abilities, specifically problem solving.

Task

A common pattern in software architecture is the publish/subscribe pattern. The pub/sub pattern is a loosely-coupled, scalable architecture that enables clients to publish and subscribe to events via a central message broker known as an event bus.

Your task is to complete the event bus implementation (found in the event-bus.ts file). The plumbing code has already been completed for you - you need only complete the 4 empty method bodies in the event-bus.ts file (on, once, off, emit).

You have as long as necessary to complete this task, however, it is expected that it should take around 30 mins to complete.

Required Outcome

Complete the 4 methods (on, once, off, emit) of the event bus implementation. Once all 4 methods are completed correctly, the following should occur (demonstrated in the Implementation section below):

  • Clicking on the Emit Event buttons in the On pane should log the event name and value of the input to the output container as many times as the buttons are clicked.
  • Clicking on the Emit Event buttons in the Once pane should log the event name and value of the input to the output container only once.

Assessment Criteria

Although it is expected that you complete this task according to the requirements above, it is not the primary criteria by which you will be assessed. The main assessment criteria for this task will relate to problem solving, code style/readability, code quality/safety, knowledge of modern web standards etc.

You will be asked questions relating to these criteria in your interview so take some time to brush up on any of these areas you feel you may be lacking.

Also, be prepared to explain your approach to this solution in your interview.

Above all, try your best, even if you can't complete the task. An incomplete but well-attempted solution is better than a poorly implemented working solution. Good luck!

Extension Task (Optional)

If you found the primary task too easy or simply wish to further demonstrate your abilities you may choose to complete any of the following:

  • Improve or restyle this page
  • Add unit tests for the event bus implementation
  • Build the implementation using a framework of your choice (Vue, Angular, Svelte etc.)
